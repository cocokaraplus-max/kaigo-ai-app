# 🦝 TASUKARU マスター設計図

**【プロダクト概要】**
介護現場の記録業務・モニタリング業務を、最新の生成AI（マルチモーダル対応）を用いて劇的に効率化し、職員の負担をゼロに近づけるための「AIアシスタント型ケース記録Webアプリケーション」。

## 🛠 1. 技術スタック（システム構成）
* **フロントエンド・サーバー**: Streamlit (Python) / Streamlit Cloud
* **データベース (DB)**: Supabase (PostgreSQL)
    * *利用テーブル*: `patients` (利用者), `records` (記録), `admin_settings` (設定), `blocked_devices` (制限), `devices` (端末管理)
* **AIエンジン**: Google Gemini API (`get_generative_model` による環境自動判別機能搭載)
* **端末認証**: `extra_streamlit_components` (CookieによるデバイスID管理)

## ✨ 2. コア機能とUX（最新状態）

### A. ログイン・セキュリティ
* **パスワードレスログイン**: 施設コード＆氏名のみで現場が使いやすい設計。
* **Cookieデバイス認証**: 初回ログイン時に推測不可能なUUID（デバイスID）を発行しブラウザに付与。`devices` テーブルへ自動登録（`device_id`, `facility_code`, `device_name`）。
* **デバイスブロック機能**: 管理者が特定の職員・端末からのアクセスを即座に遮断可能（SupabaseのRLSを無効化し、アプリ側で完全制御）。

### B. 記録入力（連続入力UI）
* **マルチモーダルAI入力**: テキスト、写真（最大5枚）、音声入力から、AIが「介護職の申し送り口調」へ自動変換。
* **連続入力特化**: 保存完了後、画面がTOPに戻らず入力欄だけがクリアされるため、複数人の記録をストレスなく連投可能。

### C. ケース記録閲覧・AI統合
* **AI統合ケース記録**: 1日分の記録をAIがワンクリックで要約。一時的な表示ではなく「AI統合記録」としてDBに公式保存（23:59:59扱い）。
* **UI/UX**: 横長にならないカード型のデザイン。確定後は「✏️ 編集」ボタンから直接テキストを修正・再保存可能。
* **日本時間完全対応**: DB上のUTC時間を、画面表示時に正確な日本時間へ変換（`parse_jst`）。

### D. モニタリング生成
* **文字数可変システム**: 100・200・300文字から出力ボリュームを選択可能。
* **支援内容の強調**: プロンプト調整により「支援内容」を積極的に盛り込む。日々の「AI統合記録」は二重集計防止のため自動除外。
