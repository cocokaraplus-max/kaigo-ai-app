🦝 TASUKARU マスター設計図（Ver. Production）
【作成日】 2026年4月7日
【プロダクト概要】 介護現場の記録業務・モニタリング業務を、最新の生成AI（マルチモーダル対応）を用いて劇的に効率化し、職員の負担をゼロに近づけるための「AIアシスタント型ケース記録Webアプリケーション」。

🛠 1. 技術スタック（システム構成）
フロントエンド・サーバー: Streamlit (Python) / Streamlit Cloud

データベース (DB): Supabase (PostgreSQL)

利用テーブル: patients (利用者), records (記録), admin_settings (設定), blocked_devices (制限), devices (端末管理)

AIエンジン: Google Gemini API (get_generative_model による環境自動判別機能搭載)

端末認証: extra_streamlit_components (CookieによるデバイスID管理)

✨ 2. 実装済みのコア機能とUX（ユーザー体験）
A. ログイン・セキュリティ（堅牢かつスムーズ）
施設コード＆氏名ログイン: パスワードレスで現場が使いやすい設計。

Cookieデバイス認証: 初回ログイン時に推測不可能なUUIDを発行し、ブラウザに付与。devices テーブルへ自動登録（device_id, facility_code, device_name）。

デバイスブロック機能: 管理者が特定の職員・端末からのアクセスを即座に遮断可能（SupabaseのRLSを解除し、アプリ側で完全制御）。

B. 記録入力（現場の連続入力を止めない）
マルチモーダルAI入力: テキストだけでなく、写真（最大5枚）や音声入力から、AIが「介護職の申し送り口調（主語や職員名を省いた簡潔な事実）」へ自動変換。

連続入力UI: 保存完了後、画面がTOPに戻らずに入力欄だけがクリアされるため、複数人の記録をストレスなく連投可能。

C. ケース記録閲覧・AI統合（記録の公式化）
AI統合ケース記録: 1日分の記録をAIがワンクリックで要約。「一時的な表示」ではなく、**「AI統合記録」という公式データとしてDBに保存（23:59:59扱い）**される。

洗練されたUIと編集機能: 横長にならないカード型のスッキリしたデザイン。確定後は「✏️ 編集」ボタンから直接テキストを修正・再保存が可能。コピーもワンクリック。

日本時間（JST）完全対応: データベース上のUTC時間を、画面表示時に正確な日本時間へ変換（parse_jst 関数）。

D. モニタリング生成（月間要約）
文字数可変システム: 100文字、200文字、300文字から出力ボリュームをラジオボタンで選択可能。

支援内容の強調: AIへのプロンプト指示により、「支援内容」に関する事柄を積極的に盛り込むよう調整済み。

ノイズ除外: 月間要約を作る際、毎日の「AI統合記録」は二重集計にならないよう自動的に計算から除外。

E. 直感的なナビゲーション
1クリック履歴ジャンプ: TOP画面の更新履歴（最新30件）の利用者名をタップすると、**「対象の利用者」と「その記録が書かれた日付」**がセットされた状態で、ケース記録画面へ1発でジャンプ（on_click コールバック使用）。

🚧 3. 乗り越えた技術的壁（デバッグと最適化の歴史）
このシステムは、以下のクリティカルな問題をすべて解決した「極めて安定した状態」である。

AIモデルの「404 NotFound」エラーの根絶:
APIバージョンやクラウド環境によって指定モデルが弾かれる問題を、**「APIに今使えるモデルを聞き出し、確実に通るモデル（gemini-1.5-flash等）を自動セットする（オートディテクト）」**仕組みで完全解決。

Duplicate Element Key エラーの解消:
Streamlit特有の「複数Cookieの同時保存クラッシュ」に対し、明示的な key パラメータを付与することで安定したログインを実現。

SupabaseのRLS（Row Level Security）の罠:
デバイス管理テーブルにデータが入らない・読めない問題を、RLSポリシーを DISABLE にすることで解決。システム側での制御に一本化した。

市販化に向けた土台は完成: すでに複数施設・複数スタッフでの運用に耐えうるアーキテクチャになっている。
